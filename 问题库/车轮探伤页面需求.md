这份清单是基于 前后端分离 的开发模式制定的。

核心分工逻辑 ：

- 你 (前端/Qt) ：负责“播放器”的搭建（HTML/JS 逻辑），确保能接收并渲染数据。
- 同事 (后端/ROS) ：负责“电视台”的搭建（数据源），确保数据通过 WebSocket 协议发出，并提供模型文件下载服务。
### 📋 任务清单 A：你需要完成的工作 (前端/Qt)
目标 ：确保 Area8 页面能正确加载 3D 引擎，并连接到指定 IP。



- 你的总预计耗时 ： 约 1 - 1.5 小时
### 📋 任务清单 B：同事需要完成的工作 (后端/ROS 2)
目标 ：提供 WebSocket 数据流、TF 坐标变换流、以及 HTTP 静态资源服务。



- 同事的总预计耗时 ： 约 3.5 - 6 小时
### 🤝 联调阶段 (双方配合)
预计耗时：1 - 2 小时

在双方都完成上述工作后，通常会遇到以下 2 个典型问题，需要坐在一起解决：

1. 模型位置飞了/散了 ：
   - 原因：URDF 中的 mesh 路径与 HTTP 服务器的目录结构不匹配。
   - 解决：调整 HTML 中的 path 前缀，或调整同事的文件目录结构。
2. 模型是白色的/无纹理 ：
   - 原因：材质文件丢失或光照设置问题。
   - 解决：通常不影响功能，如果需要美观，需检查 .dae 文件导出设置。
   建议 ：你可以先把这份清单发给同事，确认他们对 "搭建 HTTP 资源服务器" 这一步没有异议（这是最容易被忽略的一步）。
