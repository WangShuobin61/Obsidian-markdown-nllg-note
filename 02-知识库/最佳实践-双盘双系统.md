# 一、Ubuntu 挂载 Windows 的 800G NTFS 当数据盘，可行吗？

## 结论先说

> **完全可行，而且这是非常常见、非常合理的做法。**

很多人都是：

`Linux 系统盘：只放系统（100~200G） NTFS 大数据盘：代码 / 项目 / 下载 / 资料`

你的思路是**正确的工程思路**，没问题。

---

## 但有 3 个重要前提（非常关键）

### ① 必须关闭 Windows 的“快速启动”和休眠

否则：

`NTFS 会被标记为未完全卸载 → Ubuntu 挂载成只读 → 甚至损坏文件系统`

你前面已经在处理这一步，这是**必须条件**。

---

### ② 不建议把“Linux 软件”装在 NTFS 盘

可以放：

- 代码
- 文档
- 工作区
- 下载
- Git 仓库（大多数情况 OK）


**不建议放：**

`/usr /opt conda node_modules 大量软链接 docker 数据目录`

原因：

- NTFS 不支持 Linux 权限位
- 软链接/执行权限会出问题
- 性能比 ext4 慢

### 最佳实践

`128G ext4： 系统 + 软件 + 运行环境
`800G NTFS： 项目源码 + 数据 + 文档`

这就是**开发机最主流结构**。

---

## ③ 设置开机自动挂载（推荐做）

### 第一步：查看分区 UUID

在 Ubuntu 终端：

`lsblk -f`

找到：

`512G NTFS 的 UUID 800G NTFS 的 UUID`

---

### 第二步：编辑自动挂载配置

`sudo nano /etc/fstab`

加入两行（示例）：

`UUID=你的512UUID/mnt/winCntfs-3gdefaults,uid=1000,gid=100000 UUID=你的800UUID/mnt/datantfs-3gdefaults,uid=1000,gid=100000`

保存后测试：

`sudo mount -a`

**没报错就成功**，以后开机自动挂载。

# 二、从 Ubuntu 切回 Windows 后时间错乱

## 根本原因（非常经典）

因为：

`Linux 把硬件时间当作 UTC 
`Windows 把硬件时间当作 本地时间`

所以每次切系统都会：

`差 8 小时`

---

## 最标准解决方案（推荐）

### 让 Ubuntu 改成“按本地时间解释”

在 Ubuntu 执行：

`timedatectl set-local-rtc 1 --adjust-system-clock`

完成。

以后：

`两个系统时间完全一致`

---

## 为什么推荐改 Ubuntu，而不是改 Windows？

因为：

- 改 Windows 需要改注册表

- 更新可能被重置

- 企业环境不安全


而改 Ubuntu：

`一条命令，永久生效`
